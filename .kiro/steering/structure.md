# Project Structure

## Directory Layout

```
src/
├── index.ts              # MCP server entry point, transport setup, request handlers
├── tools/                # MCP tool implementations (one file per domain)
│   ├── execute-graphql.ts   # Raw GraphQL execution
│   ├── introspect.ts        # Schema introspection
│   ├── search.ts            # Entity search
│   ├── players.ts           # Player queries
│   ├── teams.ts             # Team queries
│   ├── leagues.ts           # League standings/leaders
│   ├── games.ts             # Game schedules/results
│   ├── drafts.ts            # Draft picks
│   └── reference.ts         # Reference data (leagues, seasons, etc.)
├── resources/            # MCP resource providers
│   ├── schema.ts            # Schema documentation resources
│   ├── reference.ts         # Reference data resources
│   └── guides.ts            # Usage guide resources
├── graphql/              # GraphQL client and queries
│   ├── client.ts            # GraphQL client wrapper with error handling
│   └── queries/             # (Reserved for query files)
├── generated/            # Pre-generated schema data (JSON files)
│   ├── schema.json          # Full introspection result
│   ├── queries.json         # All 321 queries with arguments
│   ├── types.json           # Key GraphQL types
│   ├── enums.json           # Enum values
│   └── reference-data.json  # Leagues, countries, positions
└── utils/                # Shared utilities
    └── formatting.ts        # Data formatting helpers

scripts/
└── generate-schema.ts    # Schema generation script (run manually)

dist/                     # Build output (generated by tsup)
```

## Code Organization Patterns

### Tool Files (`src/tools/*.ts`)

Each tool file exports:
- Tool definition object(s) with name, description, inputSchema
- Handler function(s) that implement the tool logic
- GraphQL query constants (inline or imported)

### Resource Files (`src/resources/*.ts`)

Each resource file exports getter functions that return JSON strings for MCP resources.

### Entry Point (`src/index.ts`)

- Creates MCP server instance
- Registers all tools and resources
- Sets up request handlers
- Implements dual transport (stdio/HTTP) with mode detection
- Handles session management for HTTP mode

## File Naming Conventions

- Tool files: plural domain names (e.g., `players.ts`, `teams.ts`)
- Resource files: singular category names (e.g., `schema.ts`, `reference.ts`)
- Utilities: descriptive names (e.g., `formatting.ts`, `client.ts`)

## Import Patterns

- Always use `.js` extensions in imports (ES modules requirement)
- Relative imports for local modules
- Absolute imports for dependencies
